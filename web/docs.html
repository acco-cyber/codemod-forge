CodeMod Forge

The Software Migration Engineer. CodeMod Forge analyzes an entire software repository, identifies modernization opportunities, creates a safe migration strategy, applies repository-wide transformations, validates the results, and delivers a production-ready pull request with a detailed impact report.

Not a chatbot. Not a prompt playground. Not another Copilot clone.
CodeMod Forge is a professional engineering platform where every page moves the user through a complete migration workflow.

Installation

CodeMod Forge requires Node.js 18+ and an OpenAI API key.

npm install -g codemod-forge

Or run without installing:

npx codemod-forge analyze --dir ./my-project

Or clone from GitHub:

git clone https://github.com/acco-cyber/codemod-forge.git
cd codemod-forge
npm install
npm run build

Quick Start

The fastest way to see CodeMod Forge in action:

# 1. Set your API key
export OPENAI_API_KEY=sk-...

# 2. Analyze a project
forge analyze --dir ./my-project

# 3. Run a dry-run migration
forge migrate react --from 17 --to 19 --dir ./my-project --all --dry-run

# 4. Apply approved changes
forge migrate react --from 17 --to 19 --dir ./my-project --all --yes

The --dry-run flag previews all changes without modifying files. Always run it first.

forge analyze

Scan a repository to detect frameworks, count components, find deprecated APIs, and score health.

forge analyze [options]

Options:
  -d, --dir <path>    Project directory (default: current)
  -v, --verbose       Verbose output

Output includes: framework version, language breakdown, file count, component count, deprecated API list, security issues, and health score breakdown (architecture, dependencies, security, maintainability).

forge plan

Fetch a migration guide, extract breaking changes with CodeMod Engine, and display a structured migration plan.

forge plan <library> [options]

Options:
  -f, --from <version>   Current version
  -t, --to <version>     Target version
  -d, --dir <path>       Project directory
  -g, --guide <url>      Custom migration guide URL
  --all                  Select all breaking changes

forge migrate

Run the full migration pipeline: plan, generate transforms, validate, dry-run, and apply.

forge migrate [library] [options]

Options:
  -f, --from <version>   Current version
  -t, --to <version>     Target version
  -d, --dir <path>       Project directory
  -g, --guide <url>      Custom migration guide URL
  --dry-run              Preview without applying
  --all                  Select all breaking changes
  -y, --yes              Auto-approve safe transforms
  --no-interactive       Non-interactive mode
  -v, --verbose          Verbose output

The pipeline runs through 9 agents: Repository Intelligence, Discovery, Planning, Fixture, Transform, Validator, Dry-run, Apply, and Report.

forge verify

Re-run existing transforms in dry-run mode to verify they still produce the expected results.

forge verify [options]

Options:
  -d, --dir <path>    Project directory

forge rollback

Restore project files from the most recent backup. Backups are created automatically when transforms are applied.

forge rollback [options]

Options:
  -d, --dir <path>    Project directory

Backups are stored in .codemod-forge/backup-{timestamp}/.

forge report

Display the migration report from the most recent run.

forge report [options]

Options:
  -d, --dir <path>    Project directory

Agent Workflow

CodeMod Forge uses a 9-agent pipeline where each agent has a specific responsibility:

1
Repository Intelligence Agent — Scans project, detects frameworks, counts components, finds deprecated APIs, scores health
2
Discovery Agent — Mines a migration guide (built-in or custom URL)
3
Planning Agent — Extracts structured breaking changes with CodeMod Engine
4
Fixture Agent — Prepares before/after validation cases
5
Transform Agent — Writes executable ts-morph code with detect() and apply()
6
Validator Agent — Independent CodeMod Engine review with confidence scoring
7
Dry-run Agent — Executes transforms in memory and shows diffs
8
Apply Agent — Writes approved changes with timestamped backup
9
Rollback Agent — Restores from backup with one command

Transform Engine

Transforms are generated as TypeScript files with two exported functions:

// Generated transform example
export function detect(sourceFile: SourceFile): boolean {
  // Returns true if the file contains the deprecated pattern
  return sourceFile.getDescendantsOfKind(...)
    .some(node => ...);
}

export function apply(sourceFile: SourceFile): void {
  // Applies the AST transformation
  sourceFile.getDescendantsOfKind(...)
    .forEach(node => {
      // Transform logic
    });
}

Transforms are:

  • Validated against before/after fixtures
  • Independently reviewed by CodeMod Engine
  • Executed in memory before touching disk
  • Stored in .codemod-forge/transforms/

Custom Migration Guides

Use any migration guide URL with the --guide flag:

forge migrate my-library \
  --from 1 --to 2 \
  --dir ./my-app \
  --guide https://example.com/migration-guide \
  --all --dry-run

The Discovery Agent will fetch the page, extract the migration content, and pass it to CodeMod Engine for breaking change analysis. Built-in guides are available for: React, Next.js, Tailwind CSS, Vue, Express, Prisma, and TypeScript.

CI/CD Integration

Run CodeMod Forge in non-interactive mode for CI pipelines:

FAQ

Is it safe to run on production code?

Yes. CodeMod Forge creates a full backup before applying any changes. Use --dry-run first to preview, and forge rollback to restore if needed.

Which libraries are supported?

Built-in migration guides cover React, Next.js, Tailwind CSS, Vue, Express, Prisma, and TypeScript. You can use any library by providing a --guide URL.

How does Transform Confidence scoring work?

After generating a transform, an independent CodeMod Engine review analyzes the code for edge cases and produces a confidence score (0-100%) with a written explanation. Transforms below the safety threshold are flagged for manual review.

Can I use it without an OpenAI API key?

The forge analyze command works without an API key. All other commands that generate or validate transforms require CodeMod Engine access.

How much does it cost?

CodeMod Forge itself is free and open source. You pay only for OpenAI API usage. A typical migration uses approximately $0.50-$2.00 in API calls.
